# POLY KCMS 개선 디자인 지침 — v0.2

노후화된 학원관리시스템(KCMS, ASP.NET 레거시)을 **신규 디자인 템플릿** 기반으로 전면 개편하기 위한 원칙·시스템·컴포넌트 가이드. 「학생 등록」·「학급 등록」 화면의 실제 진단을 근거로 작성.

| 항목 | 값 |
|---|---|
| 대상 사용자 | 본사 · 캠퍼스 · 강사 (3종, 권한 인지 필수) |
| 기준 디바이스 | 노트북 우선 + 패드·모바일 대응 |
| 테마 | 라이트 우선 (다크는 후속) |
| 색상 | 신규 팔레트 — Purity UI(틸)·Horizon(인디고/시안) 레퍼런스 합성 |
| 범위 | UI/UX 전면 · 타입별 템플릿 |

**v0.2 변경 요약** — ① 색상 토큰을 레퍼런스 기반으로 구체화(추상 → 실제 hex/oklch) ② 기본 템플릿 필수 요건(작업 탭·3depth 메뉴·좌측 접힘/펼침·상단 확장성·밀도) 신설 ③ 컴포넌트·IA에 반영.

## 1. As-Is 진단

두 화면 모두 같은 한계를 공유한다. **데이터·기능은 충분하나, 그것을 다룰 위계·도구·시스템이 2000년대에 멈춰 있다.**

1. **평면적 정보 위계** — 검색폼 + 대용량 테이블 + 상세 편집폼 + 메모 + 배정정보가 한 화면에 동시 노출, 모든 요소가 같은 비중이라 시선의 출발점이 없음.
2. **3중 내비게이션** — 상단 도메인 탭(8) + 좌측 아이콘 레일(13, 성격 혼재) + 하단 작업 탭. "지금 어디"를 세 군데서 읽어야 함.
3. **테이블 UX 노후** — 작은 행, 의미 불명 노란 하이라이트, 약한 정렬·필터, 가로 스크롤, 1/48 페이지. 4,770건을 빈약한 도구로.
4. **폼 UX** — 20+ 필드 섹션 없이 평면 나열, 비활성 필드 다수, 필수·선택 구분·인풋 높이 부족(터치 불가).
5. **상태를 텍스트로만** — 재학/휴학/탈퇴, 개강/폐강, 정원("3/12")이 평범한 텍스트. 상태 배지 체계 없음.
6. **액션 위계 부재 + 시각 노후** — 동등 비중 버튼 8개, 파란 그라디언트 헤더·베벨 버튼·시스템 기본 고딕.

> **핵심:** "기능이 아니라 위계·도구·시스템의 부재." 개편의 본질은 칠하기가 아니라 ① 위계 세우기 ② 내비 단일화 ③ 테이블·폼 현대화 ④ 하나의 시스템으로 묶기.

## 2. 개선 원칙

1. **위계를 만든다** — 평면을 1·2·3차로. 마스터-디테일은 좌우 동시 노출 대신 리스트 → 우측 드로어/별도 라우트로 분리.
2. **밀도는 유지하되 읽히게** — 파워유저 도구이므로 무작정 키우지 않음. 행 높이·정렬·구분선·tabular 숫자로 "스캔 가능한 고밀도". 밀도는 토글로 조정.
3. **내비게이션은 하나의 멘탈 모델** — 3중 내비 → 역할 기반 단일 GNB + 상단 컨텍스트. 잡다 아이콘은 도메인/유틸로 재분류.
4. **상태는 색으로 말한다** — 학적·운영·정원 상태를 일관된 배지·인디케이터로 표준화. 노란 하이라이트 제거.
5. **테이블은 작업 도구** — 정렬·다중 필터·컬럼 토글·고정 헤더·일괄 작업·우선순위 컬럼. 가로 스크롤은 최후 수단.
6. **폼은 흐름** — 의미 단위 섹션, 필수·선택 구분, 인풋 ≥ 40px, 인라인 검증.
7. **액션에 위계** — Primary 1개 강조, Secondary 2~3개, 나머지 오버플로(⋯), 위험 액션 분리.

## 3. 디자인 시스템 — 색상 (v0.2 핵심 개정)

### 3.1 레퍼런스에서 가져온 것 / 버린 것

| 레퍼런스 | 가져옴 (Keep) | 버림 (Drop) |
|---|---|---|
| **Purity UI** (틸/Chakra) | 틸 단일 액센트의 신선함, 네이비-온-라이트, 부드러운 카드·여백 | 너무 밝은 틸(흰 배경 대비 부족) |
| **Horizon** (인디고/시안) | 명확한 시맨틱 상태색(초록/빨강/앰버), 차트 그라디언트, 깔끔한 카드 | 보라 일색 브랜드, 곳곳의 보라 그라디언트(안티패턴·장시간 피로) |

> **합성 원칙:** 두 레퍼런스의 *명료함*(시맨틱 상태색·네이비 온 라이트·깔끔한 카드)은 채택하되, **브랜드 색은 보라를 피하고**, **그라디언트는 데이터 시각화에만** 한정한다. 레거시의 파랑 일색(Poly Blue)으로도 회귀하지 않는다.

### 3.2 토대 토큰 (중립 · 텍스트 — 고정)

라이트 테마 기준. 값은 레퍼런스 추출 기반의 출발점이며, 디자인 툴에서 oklch로 최종 정렬.

| 역할 | 토큰 | hex | oklch (근사) | 용도 |
|---|---|---|---|---|
| 배경 | `--bg` | `#F3F6FB` | `oklch(0.97 0.006 250)` | 앱 바탕 (쿨 라이트) |
| 표면 | `--surface` | `#FFFFFF` | `oklch(1 0 0)` | 카드·패널 |
| 표면2 | `--surface-2` | `#EAF0F7` | `oklch(0.94 0.008 250)` | 입력 배경·hover |
| 경계 | `--border` | `#DDE5EF` | `oklch(0.9 0.01 250)` | 구분선 (그림자보다 선 우선) |
| 본문 | `--text` | `#18243F` | `oklch(0.27 0.04 260)` | 주 텍스트 (네이비) |
| 보조 | `--text-muted` | `#64748B` | `oklch(0.55 0.03 255)` | 라벨·캡션 |

### 3.3 브랜드 액센트 — 2개 옵션 (택1)

토큰 *역할*은 동일, *값*만 교체한다. 어느 쪽도 보라/네이비-PolyBlue가 아니다.

**옵션 A · Teal (Purity 계열) — 권장**
교육·성장·신뢰의 톤. 흰 배경 대비를 위해 Purity보다 한 단계 깊게.

| 토큰 | hex | oklch (근사) |
|---|---|---|
| `--primary` | `#0EA5A0` | `oklch(0.64 0.105 192)` |
| `--primary-hover` | `#0B8F8B` | `oklch(0.58 0.10 192)` |
| `--primary-soft` | `#DCF3F1` | `oklch(0.95 0.03 192)` |
| `--on-primary` | `#FFFFFF` | — |

**옵션 B · Indigo–Cyan (Horizon 계열) — 대안**
모던 SaaS 톤. 단, Horizon 원본보다 채도를 낮춰 장시간 가독성 확보(보라→블루-인디고로 보정).

| 토큰 | hex | oklch (근사) |
|---|---|---|
| `--primary` | `#3B4FE0` | `oklch(0.52 0.18 268)` |
| `--primary-hover` | `#2E3FC4` | `oklch(0.46 0.17 268)` |
| `--primary-soft` | `#E5E8FB` | `oklch(0.93 0.03 268)` |
| 보조(차트) | `#2FB8FF` | `oklch(0.74 0.13 235)` |

> 옵션 B는 셸 방향 C(Console)와, 옵션 A는 방향 A(Workbench)와 결이 맞는다. 셸 방향이 정해지면 여기서 1개로 확정.

### 3.4 상태색 (Horizon 채택 — 배지·텍스트 전용)

브랜드 색과 **절대 섞지 않는다.** 동일 의미는 전 화면 동일 색·형태.

| 의미 | 토큰 | hex | oklch (근사) | 예시 |
|---|---|---|---|---|
| 긍정 | `--success` | `#1FA971` | `oklch(0.66 0.14 160)` | 재학 · 개강 · 승인 |
| 주의 | `--warning` | `#E8A317` | `oklch(0.76 0.14 75)` | 대기 · 미납 · 정원 임박 |
| 위험 | `--danger` | `#E0564B` | `oklch(0.62 0.18 25)` | 탈퇴 · 폐강 · 삭제 |
| 정보 | `--info` | `#2E7FE0` | `oklch(0.60 0.16 250)` | 안내 · 진행중 |

각 상태는 `*-soft`(배경 8~12% 톤) + 본색(텍스트)으로 배지 구성. 정원 "3/12"는 숫자 + 가는 진행 바(채움률에 따라 success→warning→danger).

### 3.5 그라디언트 · 엘리베이션 정책

- **그라디언트는 데이터 시각화에만** — 영역 차트 채움(브랜드색 → 투명), 막대 그라데이션 정도. UI 크롬(헤더·버튼·카드·사이드바)은 **플랫** 유지.
- **보라색 그라디언트 전면 금지** (브랜드·크롬·차트 모두).
- 그림자보다 **경계선 우선**. 떠 있는 요소(드롭다운·드로어·플라이아웃)만 약한 그림자.

### 3.6 타이포그래피 / 간격 · 형태

- **본문/UI**: Pretendard (한글·라틴 일관). *Inter/Roboto/Arial 금지.*
- **디스플레이/수치**: Space Grotesk (제목·KPI) — 가안, 셸 확정 시 고정.
- **숫자**: 정원·전화·금액·학번은 `tabular-nums` 고정폭 정렬.
- **스케일**: 12 / 13 / 14(본문) / 16 / 20 / 24 / 32.
- **그리드** 4·8pt · **라운드** 절제(입력 4~6px, 카드 8~12px; 방향 B만 더 둥글게) · **아이콘** 1.5px 스트로크, 길찾기·정보용만.

## 4. 기본 템플릿 필수 요건 (신설)

모든 화면이 올라타는 셸이 반드시 충족해야 할 구조 요건.

1. **화면별 작업 탭 (필수)** — 메뉴에서 연 화면은 콘텐츠 상단에 탭으로 누적, 전환·닫기(×) 가능(레거시 하단 탭의 현대화). 멀티태스킹 보존.
2. **3DEPTH 메뉴** — 도메인 › 중분류 › 화면 (예: 학생관리 › 학생정보 › 학생 등록). 브레드크럼도 3단.
3. **좌측 메뉴 접힘/펼침** — 접힘 = 아이콘만 + hover 시 2·3depth 플라이아웃 / 펼침 = 아이콘 + 메뉴명 아코디언.
4. **상단 메뉴(가로형) 구조 + 1DEPTH 확장성** — 1depth는 상단 가로 바. **2depth·3depth는 좌측 패널이 아니라 1depth 바로 아래에 가로로 펼쳐진다** — 1depth 선택 → 그 아래 2depth 가로 행 → 2depth 선택 → 다시 그 아래 3depth 가로 행(상단에 누적되는 다단 가로 내비). 1depth 도메인이 늘면 "더보기" 오버플로로 흡수하고, 2·3depth 행도 폭 초과 시 가로 스크롤/"더보기"로 처리해 깨지지 않게 한다.
5. **밀도 조절** — 표준/컴팩트 토글로 행·메뉴 간격 조정(데이터 최대 노출 ↔ 가독성 균형).
6. **권한 인지** — 본사·캠퍼스·강사에 따라 노출 도메인·스코프 라벨이 달라짐(강사: 학생/학급/수업 + "내 반", 본사: 전 도메인 + "전국 N캠퍼스").

## 6. 화면 타입별 적용 방향

- **리스트형** (원생 관리, 학급 등록): 필터 바(접기) → 도구 줄(검색·컬럼·일괄·내보내기) → 고정 헤더 테이블 → 하단 요약·페이지네이션. 행 클릭 = 상세 드로어.
- **상세(탭형)** (학생 상세): 요약 헤더(이름 + 상태 배지 + 핵심 메타) + 탭(기본정보·학적·출결·성적·수납·메모). 액션은 헤더 우측 위계대로.
- **폼** (학급 기본정보): 섹션 카드(운영·편성·과목·강사·비고). 요일·과목 멀티선택은 칩/토글 그룹.

## 7. 반응형 · 접근성 (품질 바닥선)

- 노트북 우선 설계 → 패드(GNB 접힘·드로어), 모바일(단일 컬럼·핵심 컬럼만)로 적응.
- 대비 WCAG AA 이상 · 보이는 키보드 포커스 · 터치 타깃 ≥ 40px.
- `prefers-reduced-motion` 존중 · 페이지 로드 애니메이션은 1회로 절제.

## 8. 적용 우선순위 · 로드맵

1. **앱 셸** (내비 + 헤더 + 작업 탭) — 방향 A/B/C 중 확정, 색상 옵션 A/B 중 1개 고정 *(진행 중)*
2. **리스트/테이블 템플릿** — 원생 관리로 대표 검증
3. **상세 / 마스터-디테일** — 드로어·탭 패턴 확정
4. **폼 템플릿** — 학급 등록·학생 등록 섹션화
5. **도메인 확장** — 입학·수업·도서·행정·직원

## 9. Do / Don't

**권장**
- 브랜드 1색(틸 또는 인디고) + 상태 4색 분리
- 네이비 온 라이트, 경계선 우선
- Pretendard + tabular 숫자
- 리스트 → 드로어 마스터-디테일
- 작업 탭 · 3depth · 권한 인지
- 그라디언트는 차트에만
